# Getting Started

\
Here is a list of things that you need to decide/prepare before you Join the network

**1. Identification of role(s)** \
Depending on your usecase, you can join the network as a Seeker, Provider or Both

**Roles You Can Play**

An entity can play various roles in OAN. The primary roles are:

**Provider Platform**: If you have scheme information/or an aggregator of all Agri-related schemes which helps the farmers/or anyone in the agri eco-system and/or you are a weather information provider  and aspire to broaden the reach to farmers or to the overall agri-ecosystem  you can join network as a Provider Platform..&#x20;

**Provider:** If you’d like to be on boarded on to an existing provider platform as one of the providers of Schemes/weather  you can join the network as a Provider.&#x20;

**Seeker Platform:** If you are an application or a platform provider that acts as an interface for farmers, or field-level workers to access information,  or aspire to become a platform that can enable access to agriculture schemes, weather advisory services available on the network, you can join Network  as a seeker platform.&#x20;

**Seeker:** If you’d like to consume the information made available by the OAN  network through any of the seeker platforms, you can join as a seeker.&#x20;

You have the flexibility to choose one of the aforementioned roles or even assume more than one role within the network, enabling you to actively engage in and benefit from the diverse opportunities provided by the network. <br>

**2. Use Case Identification**&#x20;

In this current phase of OAN,  [SCHEMES discovery](scheme-discovery.md) and  [Weather advisory](/broken/pages/u5HVYxeMDk8TDh9d4GXb) use cases are enabled.  Each of these use cases will have its own Schema and API definitions. Please visit the respective session for details .

**3. Understand the schema and transaction flows and APIs** \
For the use case that you have identified please the schema definitions and the API needs be explored well before you can build your applications.

**4. Integration of Protocol Server** \
The Beckn protocol server needs to be installed and setup before you can transact over the network

**5. If you want to onboard as a Provider Platform please refer the section Setup a Provider Platform**&#x20;

**6. If you want to onboard as a Seeker Platform please refer the section Setup a Seeker Platform**&#x20;
