# Indexing and Performance

Index settings are defined upfront, similar to defining a table schema in SQL. Ingestion is performed in parallel for performance. Search result length and token limits are tuned to optimise relevance.

The system currently retrieves top-k results (for example, top-10) per query, and the source of each result is visible to the user, improving transparency.
