# Prompt-Driven Governance & Policy Control

The behaviour of all agents and tools in the OAN AI module is governed primarily through system prompts rather than hard-coded logic.

Each agent operates with a dedicated system prompt that defines:

* Scope of responsibility
* Permitted and restricted actions
* Tool usage rules
* Language and response policies
* Safety and compliance constraints

This approach allows governance policies to evolve independently of the underlying software system. Changes to moderation rules, domain scope, or response behaviour can be implemented by updating prompts rather than modifying application code.

As a result, the AI module remains:

* Configurable across states
* Adaptable across programmes
* Governable by public institutions
* Aligned with DPI principles of transparency and auditability
