# Data Ingestion Pipeline

The Data Ingestion Pipeline in OpenAgriNet (OAN) is responsible for preparing agricultural knowledge and programme information so that it can be reliably retrieved during AI-driven user interactions. It converts raw source content from institutions such as agricultural universities, government departments, and programme teams into structured, searchable formats that can be consumed by the retrieval layer.

Unlike runtime query processing, which operates in real time for every user interaction, the data ingestion pipeline operates offline as part of system setup and periodic content updates. Its primary objective is to ensure that all knowledge made available through the AI layer is accurate, authorised, traceable, and optimised for retrieval.

The ingestion pipeline is also designed to be deployment-agnostic and reusable across states and programmes. New content sources can be onboarded using the same ingestion utilities and schema, without requiring changes to the AI orchestration or retrieval architecture. This enables rapid expansion of OAN deployments while maintaining consistency in data quality and retrieval performance.

All content processed through the ingestion pipeline is indexed with source metadata, enabling traceability and auditability of responses generated by the AI layer. At runtime, the AI module does not directly access raw documents; instead, it retrieves pre-ingested and validated content through the retrieval layer, ensuring predictable performance and grounded responses.
